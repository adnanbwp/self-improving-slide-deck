Pattern selection is workflow design — not an AI-augmented decision. Malone's finding applies to decisions made with AI. The IM/DM selects the pattern before the work begins.

Hybrid Delivery Adnan Ali · 2026
The HITL taxonomy comes from LangGraph documentation, the DEV Community HITL taxonomy, and Anthropic's Trustworthy Agents guidelines. These are engineering documentation sources, not controlled studies of IM/DM behavior. The mapping from each pattern to a specific IM/DM accountability decision is argued from functional alignment: each pattern requires a governance decision that is natively in the IM/DM's accountability space, not the tech lead's. The tech lead designs the agent. The IM/DM decides which human-in-the-loop pattern governs how that agent participates in the team's delivery system. The three IM-role entries — DoD gate design, task backlog stewardship, escalation threshold setting — are the concrete G-NEW-1 connection: these are IM/DM-specific decisions, not engineering decisions. The Malone distinction is critical for analytically-minded audiences: Malone et al. (Slide 6) establishes that adding AI to decision tasks degrades performance below the best solo performer. This is not a contradiction of the HITL argument — the IM/DM's governance role is selecting which pattern applies to which work (a workflow design decision made before the work begins), not making decisions augmented by AI within a task. The distinction matters: an IM/DM deciding "this workflow uses AI Monitors → Human Intervenes" is doing workflow architecture, not AI-augmented decision-making.
